# Student Review Portal Skill

**Skill ID**: SK-002
**Category**: Core (Base Package)
**Pricing**: Included in all tiers
**Status**: ✅ Production Ready
**Last Updated**: 2025-02-17 (v1.0)

---

## Purpose

A web page where students can review their graded essay results in detail after the teacher has graded their work. Accessed via a unique session URL or QR code (requires SK-008).

---

## Features

- ✅ Interactive essay display with error highlighting
- ✅ Sentence-by-sentence error annotations (click to reveal)
- ✅ Error list with explanations
- ✅ GEPT score display
- ✅ Revised versions (basic + advanced)
- ✅ Improvement suggestions
- ✅ Student name personalisation in title
- ✅ Mobile-friendly (students use phones)
- ✅ No login required (access via unique URL)

---

## File

**Frontend**: `student_review.html`
**Backend endpoint**: `GET /api/get-session/:sessionId`

---

## How It Works

```
Teacher grades essay (SK-001)
  ↓
Session saved to database (SK-008)
  ↓
Unique URL generated:
  student_review.html?session=abc123
  ↓
Teacher shares URL or QR code with student
  ↓
Student opens link on any device
  ↓
Page fetches session data from backend
  ↓
Interactive grading report displayed
```

---

## URL Schema

```
https://kniph.github.io/student_review.html?session={sessionId}
```

### Example

```
https://kniph.github.io/student_review.html?session=abc123def456
```

---

## API Call

```javascript
// Fetched automatically on page load
const sessionId = new URLSearchParams(window.location.search).get('session');

const response = await fetch(
  `https://railway-backend-production-55cf.up.railway.app/api/get-session/${sessionId}`
);
const data = await response.json();
// data.session contains all grading results
```

---

## Display Sections

1. **Header** — Student name (if provided), date
2. **Original Essay** — Full text with error sentences clickable/highlighted
3. **Error List** — Each error with location, issue, and explanation
4. **Revised Versions** — Basic correction + advanced model answer
5. **GEPT Score** — Score out of 5 with comment
6. **Improvement Suggestions** — Immediate / mid-term / long-term

## Usage Notes

- URLs are permanent (until session expires per data retention policy)
- No authentication — treat URL as sensitive (don't post publicly)
- Works on iOS Safari, Android Chrome, desktop browsers
- QR code for the URL is generated by SK-008
